Projet The Cool Skaters

Projet scolaire · Full-stack

Application web moderne d'e-commerce présentant différents articles sur le thème du skateboarding

Description

Ce projet a été réalisé dans le cadre de mes études lors d'un travail de groupe. Il a été développé par moi-même et deux autres étudiants

L'objectif était de concevoir un site de e-commerce en s'appuyant sur les laboratoires réalisés en cours. Le développement devait être effectué dans l'environnement IntelliJ, en utilisant les frameworks Spring, Maven, Hibernate et JUnit, avec une connexion à une base de données MySQL. L'application devait également être déployée sur un serveur Tomcat

Développement

Technologies utilisées

Ce projet a été codé en Java avec SpringBoot, utilise Hibernate en ORM pour la base de données en MySQL et implémente une sandbox Paypal et un serveur Tomcat Externe (packaging en .war)

Organisation du code

L'architecture du projet est séparée en plusieurs couches et respecte le MVC

  • Dossier model dans skateSite/model qui répertorie les modèles basés sur les entités de la base de données MySQL
  • Dossier controller dans skateSite/controller qui répertorie les controlleurs qui font le lien entre les modèles et les vues
  • Dossier jsp dans webapp/WEB-INF/jsp qui contient les fichiers jsp qui sont des templates HTML utilisés en Spring pour l'affichage des vues et dossier css dans webapp/css où l'on retrouve les différentes feuilles de style CSS des templates jsp

L'utilisation d'Hibernate a également été organisée selon une structure définie dans skateSite/dataAccess

  • Dossier dao dans skateSite/dataAccess/dao consistant la couche business qui définit quelles actions sont possibles via un interface et son implémentation
  • Dossier entity dans skateSite/dataAccess/entity définissant les entités présentes dans la base de données pour les requêtes de Hibernate
  • Dossier repository dans skateSite/dataAccess/repository contenant les repository remplis automatiquement par Hibernate selon le nom et attributs donnés des méthodes déclarées
  • Dossier util dans skateSite/dataAccess/util dans lequel se trouve le DozerBeanMapper utilisé dans ProviderConverter.java

Connection

L'application est composée de plusieurs pages, et la première que l'on voit est celle-ci : la page de connection

Bien qu'il n'est pas nécessaire d'être connecté pour consulter le site, il reste néanmois recommandé de le faire car pour pouvoir passer commande, il faut être connecté

Cette page est assez simple, un formulaire de connection demandant d'insérer un login et un mot de passe afin de se connecter ou alors de créer un compte

Page Accueil

Ensuite vient la page d'accueil, que l'on se soit connecté ou pas, elle reste accessible et comporte plusieurs éléments

Mais d'abord parlons du header et du footer :

Le footer contient un lien vers un page courte qui décrit l'entreprise qui utiliserait ce site d'e-commerce (entreprise factice pour le projet)

Le header, quant à lui, est bien plus intéressant, il permet de changer la langue du site selon le drapeau cliqué (la langue sélectionnée par défaut est la locale donnée par le navigateur), ensuite il y a un bouton Home qui permet de revenir à l'accueil

Après cela, il y a un lien vers le panier du client et enfin un texte permettant de se connecter ou déconnecter selon la connection de l'utilisateur

Quant à la page d'accueil, elle comporte le nom du site et toutes les catégories d'articles disponibles, le style de la page n'est pas très recherché mais ce n'était pas là le but du travail et cela se voit malheureusement sur deux pages du site dont celle-ci

À partir de cette page, nous pouvons accéder aux différents articles directement, comme le voudrait un bon site d'e-commerce

Une fois arrivé sur une catégorie on peut voir directement les articles, leur nom, description et prix, ces noms sont traduits selon la langue choisie

Si le client tombe sur un article qui l'intéresse, il peut cliquer dessus et aller voir les détails du produit

Article

Une fois sur la page de détails de l'article, l'utilisateur peut voir des informations en plus telles que la marque, la/les couleur(s), les dimensions/taille et enfin voir si le type de modèle

Mais cette page permet surtout d'ajouter l'article dans le panier via un bouton en bas de page

Panier

Après avoir ajouté les articles désirés, le client peut aller dans son panier où il retrouvera lesdits articles

Le panier est accessible même sans article ajouté comme montré ci-dessous mais il est aussi accessible si l'utilisateur n'est pas connecté, cela est possible car le panier est gardé en session

Par contre, pour pouvoir continuer à partir de là, il faut se connecter, d'ailleurs si un utilisateur non connecté essaie de commander, il sera redirigé vers la page de connection

Paypal

Nous voici à la dernière étape, celle du paiement

Ici ce n'est pas une vraie page paypal, seulement la sandbox, car ce projet n'a pas été créé dans un esprit de production et aussi car c'était une partie des consignes

Après s'être connecté et avoir payé la commande, le client est redirigé vers une page du site lui disant que le paiement a été effectué et à ce moment là, la commande est insérée en base de données et permettra par la suite de faire un historique des commandes

Conception et design

Avant de créer l'application web, il a fallu analyser les besoins d'un tel projet, dans cette optique, nous avions réalisé un diagramme de classe qu'on retrouve ci-dessous

En premier, il y a le Client, qui représente l'utilisateur du site, toutes les données présentes ici sont demandées à la création d'un nouveau compte

Celui-ci n'a qu'un seul lien, au niveau de la Commande qui représente une commande passée par un client (une fois le panier vidé après la confirmation d'une commande, les infos sont reprises ici)

La Commande est datée (avec heure) et contient plusieurs lignes (des LigneCommande) qui précisent le nombre d'articles commandés et si une promotion a été appliqué sur ce produit au moment de l'achat

Il y a bien sûr une classe Produit qui reprend toutes les informations d'un article sauf celles qui peuvent être traduites car cela se trouve dans une autre classe : la classe Traduction

La classe Traduction a un lien avec le produit qu'il traduit et un autre lien avec le Langage dans laquel il traduit
